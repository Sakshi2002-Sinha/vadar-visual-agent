# VADAR Visual Agent

A **VADAR-inspired visual agent** that performs 3D spatial reasoning on regular 2D images.
Give it a picture of a room and ask *"Is the green chair further away from the camera than the wooden table?"* – and the agent figures out the answer by combining state-of-the-art vision models with GPT-4-powered agentic code generation.

---

## Features

- **Object detection** with [DETR (facebook/detr-resnet-50)](https://huggingface.co/facebook/detr-resnet-50)
- **Monocular depth estimation** with [DPT-Large (Intel/dpt-large)](https://huggingface.co/Intel/dpt-large)
- **Panoptic segmentation** with [DETR Panoptic](https://huggingface.co/facebook/detr-resnet-50-panoptic)
- **Agentic code generation** – GPT-4 writes custom Python code on the fly to answer each spatial question
- **Evaluation benchmark** – saves generated programs, answers, and visual traces for every question
- **Jupyter notebook** for quick interactive demos

## Quick Start

### 1 · Clone the repository

```bash
git clone https://github.com/Sakshi2002-Sinha/vadar-visual-agent.git
cd vadar-visual-agent
```

### 2 · Create and activate a virtual environment

```bash
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
```

### 3 · Install dependencies

```bash
pip install -r requirements.txt
```

### 4 · Set your OpenAI API key

```bash
# Copy the example and fill in your key
cp .env.example .env
# Then edit .env and replace sk-your-api-key-here with your real key
```

Or export it directly:

```bash
export OPENAI_API_KEY="sk-..."
```

### 5 · Verify the setup

```bash
python verify_setup.py
```

## Usage

### Python API

```python
from vadar_agent import VADARAgent
import os

agent = VADARAgent(os.getenv("OPENAI_API_KEY"))

result = agent.answer_question(
    "Is the green chair further away from the camera than the wooden table?",
    "path/to/room.jpg"
)

print(result["answer"])   # e.g. "Yes – the chair (depth 0.72) is farther than the table (depth 0.41)"
print(result["status"])   # "Success"
print(result["code"])     # the Python code GPT-4 generated to arrive at the answer
```

### Jupyter Notebook

```bash
jupyter notebook demo_notebook.ipynb
```

The notebook walks you through every step interactively, with inline visualisations of the depth map and detected objects.

### Evaluation Benchmark

```bash
python evaluate_benchmark.py
```

To use your own test cases, pass a JSON file:

```bash
python evaluate_benchmark.py --test-cases my_test_cases.json
```

The JSON file should follow this schema:

```json
[
  {
    "sample_id": "room_001",
    "image_path": "sample_images/room_001.jpg",
    "questions": [
      "Is the chair farther from the camera than the table?",
      "Which object is closest to the camera?"
    ]
  }
]
```

Results are written to `benchmark_results/` (configurable with `--output-dir`):

| File | Description |
|------|-------------|
| `evaluation_results.json` | Full Q&A results for every sample |
| `summary_report.json` | High-level statistics |
| `trace_<id>_q<n>.png` | Visual trace: image + boxes + depth map + answer |
| `code_<id>_q<n>.py` | The Python code generated by GPT-4 |

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `ImportError: No module named 'openai'` | Run `pip install -r requirements.txt` |
| `OPENAI_API_KEY not set` | Copy `.env.example` → `.env` and fill in your key |
| Slow first run | Vision models (~400 MB) are downloaded from Hugging Face on first use |
| `CUDA out of memory` | Set `CUDA_AVAILABLE=0` to force CPU mode |
| `RuntimeError` from transformers | Ensure `torch>=2.0.0` is installed |
| Notebook kernel dies | Restart the kernel and re-run cells one by one; check RAM usage |
